<div class="diagram-container" id="consent-flow">

<div class="diagram-header">3. Consent Capture Flow</div>

<div class="diagram-description">

<div class="diagram-content">

<pre class="mermaid">

autonumber

participant Staff

participant D365 as Dynamics 365

participant Dyn as Dynamics API

Staff->>D365: Open Contact record

D365->>UI: Load embedded panel

UI->>API: GET /api/patients/:id/consents

API->>DB: Query consent status

DB-->>API: No active consent

API-->>UI: {status: "missing"}

UI-->>D365: Show yellow warning badge

Staff->>UI: Click "Request Consent"

UI->>UI: Open consent form modal

Staff->>UI: Select template + channels

Staff->>UI: Hand tablet to patient

Note over UI: Patient Interaction

UI->>UI: Patient reviews terms

UI->>UI: Patient selects: Email ✓, SMS ✓

UI->>UI: Patient draws signature

UI->>UI: Patient clicks "I Agree"

UI->>API: POST /api/consents

Note right of API: Payload:<br/>{patient_id, template_id,<br/>channels: ["email","sms"],<br/>signature_data: {svg, type},<br/>ip, user_agent, timestamp}

API->>DB: BEGIN TRANSACTION

API->>DB: INSERT consent record

API->>DB: INSERT audit_log + SHA-256 checksum

API->>DB: COMMIT

API->>Queue: Queue: Generate PDF

API->>Queue: Queue: Sync to Dynamics

API-->>UI: {success: true, id: "uuid"}

Queue->>API: Generate signed PDF

API->>DB: Store PDF path

Queue->>Dyn: PATCH /contacts(id)

Note right of Dyn: {ch_marketing_email: true,<br/>ch_marketing_sms: true,<br/>ch_consent_date: "2026-02-03"}

Dyn-->>Queue: 200 OK

Queue->>DB: Update sync_status = "synced"

UI-->>D365: Refresh panel

D365-->>Staff: Show green checkmark ✓

<!-- 6. State Machine -->

<div class="diagram-container" id="state">

<div class="diagram-header">6. Consent Status State Machine</div>

<div class="diagram-description">

Valid state transitions for consent records. Once revoked or expired, a new consent must be captured (cannot reactivate old record).

</div>

<div class="diagram-content">

<pre class="mermaid">

stateDiagram-v2

[*] --> pending: Consent Requested

pending --> active: Patient Signs

pending --> [*]: Request Expires (7 days)

active --> revoked: Patient/Staff Revokes

active --> expired: Expiration Date Reached

revoked --> [*]: Terminal State

expired --> [*]: Terminal State

note right of pending

Waiting for patient

to complete form.

Token sent via email

or opened in-person.

end note

note right of active

Valid consent.

Synced to Dynamics.

Allows marketing contact.

end note

note right of revoked

Manually revoked by

patient or staff.

Audit logged with reason.

Dynamics fields cleared.

end note

note right of expired

Auto-expired based

on template settings.

Requires new consent

to resume contact.

end note

</pre>

</div>

rect rgb(232, 245, 233)
